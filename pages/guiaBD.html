Guía de Bases de Datos

Ventajas

  • Independencia física: La forma de almacenar los datos no influye en su manipulación.
  • Flexibilidad y seguridad: Ofrecen distintas vistas según los usuarios y restringen el acceso.
  • Uniformidad: Las estructuras lógicas siempre tienen una única forma conceptual.
  • Menor redundancia y mayor integridad: Se reduce la repetición de datos y se minimizan incoherencias.

Desventajas

  • Necesidad de personal calificado.
  • Requerimientos adicionales de hardware.
  • Falta de rentabilidad a corto plazo.

Conceptos Básicos

  • Objetivo: Organizar y almacenar datos para su fácil manejo.
  • Una base de datos es un sistema formado por una colección de datos almacenados y categorizados de distintas maneras, pero que comparten algún tipo de vínculo que permita el acceso directo a ellos, así como sus relaciones.
  • DBMS/SGBD: Sistema de gestión de bases de datos.
  • Propósito del DBMS/SGBD: Proporcionar herramientas eficientes para manipular la información de manera rápida, fácil y segura.
  • Funciones: Añadir, modificar, eliminar datos, consultar información y generar reportes.
  • Un DBMS/SGBD debe: Crear bases de datos, mantener información y explotarla.

Definición y Componentes de una Base de Datos Relacional

Una base de datos relacional almacena información en tablas. Estas están formadas por campos y registros.

  • Registro: Información almacenada en cada campo.
  • Campo: Dato específico de un registro con nombre, tamaño y dirección.

Tipos de Base de Datos

  • Plana: Formada por una sola tabla.
  • Relacional: Constituidas por múltiples tablas organizadas.

Tipos de Claves

  • Clave principal: Identificador único para cada registro en una tabla.
  • Clave externa: Campo en una tabla que hace referencia a una clave principal en otra tabla.
  • Clave compuesta: Grupo de campos que identifican de manera única un registro.

Normalización

Proceso para eliminar redundancias e inconsistencias.

  • Objetivos de la normalización:
    • Evitar redundancia de datos.
    • Evitar problemas de actualización.
    • Proteger la integridad de los datos.
  • Primera Forma Normal (1FN):
    • Eliminar grupos repetitivos en las tablas.
    • Crear una tabla separada para cada grupo de datos relacionados.
    • Identificar cada grupo de datos con una clave primaria.
  • Anomalías en bases de datos:
    • Inserción: Ocurre cuando se inserta una nueva fila sin respetar las dependencias funcionales.
    • Modificación: Se da cuando se modifican columnas con datos redundantes solo en algunas filas.
    • Eliminación: Ocurre cuando se eliminan filas que contienen datos redundantes y se pierde información clave.
